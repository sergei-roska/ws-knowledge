# Workflow: Refactor

## Objective
The goal of this workflow is to improve code quality, readability, and maintainability without changing its external behavior.

## Steps
1. **Analyze:** Carefully review the provided code or file.
2. **Identify Issues:** Find any 'code smells' (e.g., duplication, long methods, complex logic).
3. **Propose Changes:** Describe the refactoring strategy.
4. **Implement:** Apply surgical changes using available tools.
5. **Verify:** Ensure tests still pass (if any exist) and the code adheres to project standards.

## Principles
- DRY (Don't Repeat Yourself)
- SOLID principles
- Clean Code (descriptive naming, small functions)
- Consistent style
