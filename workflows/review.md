# Workflow: Code Review

## Objective
Provide a thorough review of the code to ensure high quality, performance, and adherence to best practices.

## Steps
1. **High-level View:** Explain what the code does.
2. **Quality Check:** Look for bugs, edge cases, and potential security issues.
3. **Performance:** Identify bottlenecks or inefficient algorithms.
4. **Readability:** Check naming, structure, and comments.
5. **Summary:** Give a final verdict (LGTM or changes requested).

## Focus Areas
- Error handling
- Type safety
- Modularity
- Testability
